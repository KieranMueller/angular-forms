  • Navigate to the 'Saving data' page at the top, fill out

What's going on here?

  • On the 'Saving data' page, you entered information which
    we saved to local storage
  • When this component is created, within the ngOnInit()
    lifecycle hook, we are retrieving that data from local storage
  • Remember, local storage can only store strings, so we utilize
    JSON.parse() for our non-string variables
  • Notice we are doing a bit work to ensure that our variables
    are being set to truthy values, we do not want to assign a
    variable to a local storage call for a key that does not exist
  • I would advise against using the typescript exclamation mark
    (non-null operator), it can lead to unexpected behavior if
    you try to fetch keys from local storage that do not exist
  • This data can now be accessed anytime, at any place in the
    application, without the need for an HTTP request to a backend server
